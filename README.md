# NakUrut Product Requirement Document

This is a Massage booking Mobile application that can be used to search, book, and review massage services offered by a Partner or a Company.

## Problem Statement: 
In Malaysia, massage services is a common practice and business. Customer would either request a masseur/professional therapist to come to their place or the other way around to get the service. There is no proper platform for the customer to find, contact, and book appointments with the masseur. 

## Objective: 
To provide a centralized massage booking platform for customers to find and book massage services from masseur and also commercial massage business.

There are 3 persona i.e. user for this app

* Customer (The one enquiring the service)
* Partner (The one offering massage service)
* Admin - Web (The one administrating the application)

The expected timeline to complete the Minimum Viable Product (MVP) is about 6 months, which is 2 months for each persona's platform.

The teck stack used is: 

* Frontend: React-Native
* Backend: Express.js
* Database: MySQL
* Hosting: Hybrid (Firebase + Railway)
* APIs: Auth, Booking, Payment


Scope: There are a few functionalities that the customer, partner, and admin should have in the MVP.

### Customer
* Register as user to the app
* Login to their profile account
* Viewing the dashboard to show booking history, current location, suggested type of services
* Searching for nearby masseur/professional therapist in a certain radius
* Book an appointment
* Rate service to the partner
* Change the language of the app (BM)

### Partner
* Register as Partner to the app
* Login to their profile account
* Viewing the dashboard to show service and customer history, current location, etc
* Accept an appointment
* Rate the customer
* Change the language of the app (BM)

### Admin
* Monitor bookings, number of users and partners, etc
* Activate/Deactivate users
* Validate registering users/partners
